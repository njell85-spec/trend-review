# Trend Review — 에이전트 메모

## 사용자
- 호칭은 항상 **PeterJ**, **존댓말** 유지.
- 작업은 대부분 **리모트 컨트롤(안드로이드 폰/태블릿)**. 데스크탑 사용은 어려움.

## 저장소
- `Trend_Review`(대문자) = 예전 데스크탑(VS Code) 원본.
- `trend-review`(소문자, **현재 작업 home**) = 리모트 컨트롤 연동용. 신규 작업은 여기서.

## 작업 습관
- 작업 **마무리 시 commit + push**, 시작 시 `git pull` 로 데스크탑과 동기화(허브=GitHub).

## 세션 리추얼 (모델 불문 — Opus 세션도 동일 품질을 내기 위한 장치)
- **시작**: `docs/HANDOFF.md` 필독 → §1 "먼저 읽을 파일" 순서대로 맥락 복원.
  §3 확정 결정은 되묻지 않는다. §4 불변식(데일리 코어 무영향)을 깨는 변경 금지.
- **마무리**: HANDOFF §5(완료 상태)·§8(진행 중이던 것) 갱신 + commit/push —
  다음 세션은 이 파일 하나로 이어받는다.
- **규모 있는 작업 4규칙**: ① 계획 문서 먼저(`docs/superpowers/plans/`)
  ② 작은 단위 커밋 ③ 병합 전 병렬 서브에이전트 리뷰(실버그 위주, 가능하면 상위 모델)
  ④ 증거(명령·출력) 없는 완료 주장 금지.
- 반복 판단·검증 작업은 가능하면 **Actions 워크플로우로 고정**(예: `video-sample.yml`,
  `ci.yml`) — 모델 재량 개입을 줄여 세션 품질 편차를 흡수한다.

## 품질 게이트 (이 프로젝트 전용)
- 리포트·발송·대시보드 관련 작업 전 **REPORT_SPEC.md 필독** (단일 기준 문서).
- 대시보드·메시지 포맷 변경 시 push 전 **/preview 스킬로 미리보기 승인** 받기.
- 커밋 전 **`npm run spec-lint` 통과** 확인 (CI 앞단에서도 강제됨).

> 세부 목표·운영 방침은 아직 확정 전이라 여기 못 박지 않음. 대화로 조율하며,
> 현재 작업 중인(확정 아님) 스펙은 REPORT_SPEC.md 참고.
