④ Harness / Infrastructure Layer (전 Agent 공통 적용)
🔌 CircuitBreaker
API 연속 실패 시 자동 차단
PubMed-API · Claude-API 각각 독립
CLOSED (정상) → failureThreshold: 3 초과
→ OPEN (차단, 30s) → 자동 HALF-OPEN
→ 성공 시 CLOSED 복귀
🔁 RetryHelper
지수 백오프 자동 재시도
maxAttempts: 3 · baseDelay: 2–3s
attempt 1: 즉시
attempt 2: ~3s 후
attempt 3: ~6s 후
이후: CircuitBreaker로 전달
💾 Cache (파일 기반)
output/cache/*.json · TTL 24–72h
키 버전으로 스키마 변경 대응
pmids_{query}_{dates} — PubMed 검색 articles_{pmids} — 논문 데이터 scores_{pmids} — Claude 점수화 pico_v2_{pmid} — PICO (한글 포함) ft_v1_{pmid} — Full-text

→ 상호 통신 → 동적 실행

⑦ 실제 데이터 흐름 요약
PubMed (50편) → 검증 (품질 필터) → Claude 점수화 (1–10) → Top-3 선별 → PMC Full-text (OA 논문) → Unpaywall (구독 논문) → Claude PICO 분석 (EN+KO, full-text 포함) → HTML Dashboard
이번 테스트 결과: JAMA Network Open 2편 → PMC full-text (26k + 23k chars, figures 포함)  |  Elsevier 1편 → abstract only
