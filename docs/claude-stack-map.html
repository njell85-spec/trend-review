클로드 코드(클코)가 뭔가요?

클로드 코드는 AI(클로드)에게 파일 읽기·수정, 터미널 명령, git(코드 기록 관리) 같은 실제 작업 도구를 쥐여 주는 프로그램입니다. 채팅만 하는 AI와 달리, 지시하면 코드를 직접 고치고 저장하고 배포까지 이어갑니다. PeterJ는 이것을 폰·태블릿(claude.ai/code)에서 원격으로 부립니다. 🏥 비유 — 병원에 비유하면: PeterJ는 최종 결정하는 지도전문의, 클로드는 담당 주치의, 아래 나올 서브에이전트들은 협진 의뢰입니다. 이 문서 전체에서 이 비유를 유지합니다.

2장 · 설정 3층

지침과 도구는 어디서 오나

클로드가 세션을 시작하면 세 층의 설정이 겹쳐져 행동 규칙과 도구가 정해집니다. ① 어느 프로젝트에서든 적용되는 전역층, ② 그 프로젝트(repo = 코드 저장소)에만 적용되는 프로젝트층, ③ 접속 환경이 기본 제공하는 세션층. 🏥 병원 전체 규정(전역) → 병동별 프로토콜(프로젝트) → 그날 배정된 진료실 장비(세션) 순으로 겹쳐지는 것과 같습니다.
① 전역층 모든 프로젝트 공통 · 원본은 trend-review 저장소, 세션 시작 때 자동 복사
PeterJ가 직접 만든 층입니다. 어떤 repo에서 작업하든 똑같이 적용됩니다.
CLAUDE.md 직접 제작 전역 업무 수칙 — 존댓말, "증거 없는 됐습니다 금지", 커밋 전 셀프리뷰 체크리스트
peterj-session-start.sh 직접 제작 세션 시작 자동화 스크립트 (3장 참고)
stop-hook-git-check 직접 제작 작업을 저장(커밋·푸시) 안 하고 끝내려 하면 막는 안전장치
superpowers 플러그인 일하는 순서를 강제하는 기성 스킬 묶음 (4장에 전체 목록)
codex-debate 스킬 직접 제작 클코덱스 토론·설계토론 절차서

② 프로젝트층 repo마다 다름 · 아래는 EMR_Assist_v1 기준
그 프로젝트에서만 지켜야 할 계약과 그 프로젝트 전용 도구.
AGENTS.md 직접 제작 프로젝트 헌법 — 절대 깨면 안 되는 계약 4조(예: 환자 데이터 비저장), 코드 지도, 세션 인계 일지(Handoff). 클로드와 codex가 같이 읽습니다.

3장 · 자동화 (훅)

사람이 안 시켜도 도는 것들

훅(hook) = 특정 순간이 오면 자동으로 실행되는 규칙입니다. "세션이 켜지면", "push하려고 하면", "끝내려고 하면" 같은 순간마다 검사가 돌아서 사람이 깜빡해도 시스템이 잡아 줍니다. 🏥 스탠딩 오더 — 입원하면 자동으로 나가는 기본 처방 세트와 같습니다.
세션 시작
작업 환경 자동 준비 직접 제작
최신 코드 받기(안전할 때만) → superpowers 설치 확인 → 전역 지침과 자작 스킬을 원본(trend-review)에서 복사 → git 신원 설정
작업 중
품질 게이트 직접 제작 (오늘 추가)
클로드가 git push(코드 올리기)를 하려는 순간 자동으로 전체 테스트(211개)를 돌리고, 하나라도 실패하면 올리기 자체를 차단
턴 종료
저장 안 하고 못 끝냄
커밋(저장)·푸시(업로드) 안 된 변경이 남아 있으면 종료를 막고 마무리를 요구
세션 마무리 관례
인계 일지(Handoff) 갱신
한 일·보류한 결정·다음 할 일을 AGENTS.md에 기록 — 다음 세션의 클로드가 첫 턴부터 이어받습니다 🏥 근무 교대 인계장 — 다음 당직의가 바로 진료를 잇게.
