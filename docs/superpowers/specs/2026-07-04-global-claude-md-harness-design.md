## 4. [과제 C] 격차를 메우는 3계층 전략

원칙 세 가지:
1. **재발 방지는 기억이 아니라 시스템에**: "다음엔 조심"이 아니라 훅·스크립트·스킬로 강제한다.
2. **결정적 도구가 할 수 있는 검증은 LLM에 맡기지 않는다**: 금지어·채널 정합·시크릿 노출은 grep/lint가 더 확실하다 (LLM 배치 스코어링→MetadataScorer 전환에서 이미 검증된 교훈).
3. **약한 모델일수록 절차를 더 세게**: 지시는 모델 공통, 검증 게이트는 하네스가 모델과 무관하게 강제.

- **계층 1 — 지시(전역 CLAUDE.md)**: 모델이 '아는' 것을 '하게' 만든다 (§5).
- **계층 2 — 하네스(훅·스크립트·스킬)**: 모델 기억에 의존하지 않는 결정적 게이트 (§6).
- **계층 3 — 위임(서브에이전트)**: A6(추론 깊이)은 커밋 전 리뷰를 상위 모델 서브에이전트로 위임해 우회 (§6 H5).

## 6. [산출물 2] 하네스 개선안

### H1. 부트스트랩-원본 디버전스 해소 [즉시 · 난이도 하]
**발견된 문제 [사실]**: `env-bootstrap.sh`가 전역 CLAUDE.md 내용을 **heredoc으로 내장**하고 있어, 이미 저장소 원본과 내용이 다릅니다(부트스트랩 판에는 "작업 습관" 섹션이 있고 원본 `global-CLAUDE.md`에는 없음). 원본이 "이 파일이 원본입니다"라고 선언하지만 실제로는 2원화.
**개선안**: env-bootstrap이 heredoc 대신 저장소(public)에서 원본을 받아오게 한다.
```bash
# env-bootstrap.sh 의 (3) 전역 지침 적용 부분 교체안
RAW="https://raw.githubusercontent.com/njell85-spec/trend-review/main/.claude/global-CLAUDE.md"
curl -fsSL "$RAW" -o "$HOME/.claude/CLAUDE.md" 2>/dev/null \
  || true  # 실패 시 apply-global-md.sh(저장소 열람 시)가 복구
```
CLAUDE.md 개정이 승인되면 `global-CLAUDE.md` 한 곳만 고치면 모든 경로에 전파됩니다.

### H2. `/preview` 스킬 — push 전 모바일 미리보기 [즉시 · 난이도 중하 · 효과 최대]
B4(UI 왕복 12+ PR)를 직접 제거. 리모트 환경에 Chromium+Playwright가 프리설치되어 있어 바로 구현 가능.
- `.claude/skills/preview/SKILL.md`: "대시보드·메시지 포맷 변경 후 push 전에 반드시 실행" 트리거.
- 동작: ① `index.html`(또는 생성 결과물)을 뷰포트 390px(폰)·800px(태블릿)로 스크린샷 ② 카톡 포맷 변경 시 `KakaoNotifier.buildReportMessages()` 실제 출력 텍스트 표시 ③ 이미지·텍스트를 사용자에게 전송하고 승인 후 push.
- 스크립트 골격(`scripts/preview.mjs`):
```js
import { chromium } from 'playwright';
const b = await chromium.launch({ executablePath: '/opt/pw-browsers/chromium' });
for (const [name, width] of [['phone', 390], ['tablet', 800]]) {
  const p = await b.newPage({ viewport: { width, height: 900 } });
  await p.goto('file://' + process.cwd() + '/index.html');
  await p.screenshot({ path: `output/preview-${name}.png`, fullPage: true });
}
await b.close();
```

### H3. `spec-lint` — 스펙 드리프트의 결정적 검사 [즉시(스크립트) · 난이도 중]
B1을 grep 수준에서 기계적으로 차단. LLM 기억에 의존하지 않음.
- `scripts/spec-lint.mjs` 검사 항목 (REPORT_SPEC에서 도출):
  - 금지어: `Top 3`, `최근 30일`, `40~50편`, `🥇`(카톡 빌더 내) 등이 4대 산출 경로(GitHubPublisher·NotificationAgent·KakaoNotifier·index.html)에 등장하면 실패.
  - 정합: `180일/300편/1편` 표기가 3채널에서 일치하는지, 카톡 빌더에 대시보드 링크가 존재하는지.
  - 안전: `output/selected_*.json`이 gitignore에 걸려 있지 않은지, 로그 문자열에 `TOKEN`류 변수가 직접 보간되지 않는지(정규식 휴리스틱).
- 연결 지점(둘 다 가능, 병행 권장):
  - **CI 스텝**(daily-review.yml 앞단) — 무인 실행도 보호. [즉시]

### H4. 전역 개인 스킬 배포 [실험 · 난이도 중]
